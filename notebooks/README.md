# Notebooks

This directory is reserved for lightweight, reproducible exploratory notebooks that inspect generated artifacts from the scripted pipeline. Notebooks should not replace the command-line workflow or become the source of manuscript evidence.

## Notebook role

Notebooks may be used for:

- inspecting generated stream manifests;
- reviewing embedding manifests;
- checking TDA feature distributions;
- visualizing persistence-entropy and distance-to-baseline summaries;
- comparing internal randomness diagnostics with topological summaries;
- drafting exploratory figures before they are promoted to scripted outputs.

## Notebook boundary

Notebooks should not be used to make final claims unless the same result is also generated by a script and registered in `docs/evidence_register.md`.

A notebook observation becomes public-facing evidence only after it is:

1. converted into a reproducible script or pipeline stage;
2. written to a documented table, figure, or log path;
3. checked by artifact coherence and consistency validation;
4. registered in `docs/evidence_register.md`.

## Suggested notebook sequence

Create notebooks in this order when generated artifacts are available:

1. `01_manifest_audit.ipynb`  
   Inspect stream counts, condition labels, replicate coverage, byte counts, and SHA-256 digest coverage.

2. `02_tda_feature_review.ipynb`  
   Review H0/H1 feature distributions by condition, embedding, and replicate.

3. `03_distance_to_baseline_review.ipynb`  
   Inspect distance-to-`sha256_seeded_baseline` tables and identify which results are stable enough to script.

4. `04_randomness_comparison_review.ipynb`  
   Compare internal randomness diagnostics with TDA summaries and external-test availability status.

5. `05_claim_trace_review.ipynb`  
   Cross-check candidate manuscript claims against registered artifacts in `docs/evidence_register.md`.

## Reproducibility note

The primary reproducible workflow remains the segmented command-line pipeline documented in the repository README. Notebooks should read existing CSV artifacts and avoid silently regenerating data with undocumented parameters.
