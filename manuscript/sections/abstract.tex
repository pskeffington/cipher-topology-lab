\begin{abstract}
This study develops a reproducible computational pipeline for evaluating whether persistent-homology summaries of ciphertext-derived point clouds provide useful auxiliary diagnostics for symmetric-cipher output randomness. The current pre-release design compares AES-128 CTR deterministic-output conditions, a seeded SHA-256 expansion reproducibility baseline, operating-system cryptographic random output as a non-deterministic sensitivity condition, and intentionally weak generator controls. Ciphertext and generator streams are locally generated from documented keys, nonces, plaintext patterns, seeds, replicate indices, and software versions, avoiding dependence on restricted datasets. Byte-pair and sliding-window embeddings are used as the configured point-cloud transformations, with cubical encodings retained as an explicitly configured extension path. Persistent-homology features are then compared with conventional randomness diagnostics. The study does not propose a new cryptanalytic attack or claim that topological summaries can certify cryptographic security. Its contribution is a transparent, reproducible framework for testing whether topological data analysis can supplement established statistical diagnostics in cryptographic implementation review.
\end{abstract}
