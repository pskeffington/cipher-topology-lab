# Thesis Scaffold

## Working thesis frame

Persistent-homology summaries can serve as reproducible auxiliary diagnostics for detecting deliberately structured weak-control byte streams under registered byte-pair and sliding-window embeddings. Across the current evidence base, deliberately structured weak controls separate from a SHA-256 seeded deterministic baseline, while AES-CTR, OS CSPRNG, and xorshift32 do not show comparable top-ranked separation under the tested design.

This is a diagnostic and reproducibility thesis. It is not a cryptanalytic attack thesis.

## Manuscript architecture

The LaTeX manuscript should remain modular:

- `manuscript/main.tex`: root document only.
- `manuscript/preamble/packages.tex`: package imports and document-level settings.
- `manuscript/preamble/commands.tex`: semantic commands for conditions, baselines, embeddings, and claim-boundary language.
- `manuscript/sections/`: chapter/section content.
- `manuscript/references.bib`: bibliography database.

This structure keeps formatting, semantic names, and evidence interpretation separated from prose.

## Object-oriented LaTeX practices

Use semantic commands for repeated objects rather than hard-coded text. For example, condition names, baseline names, backend names, and evidence-register paths should be defined once in `manuscript/preamble/commands.tex` and reused across sections.

Use `\label{}` consistently for every section, table, and figure. Labels should follow predictable prefixes:

- `sec:` for sections.
- `subsec:` for subsections.
- `tab:` for tables.
- `fig:` for figures.
- `app:` for appendix sections.

Tables should live close to the prose that interprets them, but final camera-ready versions may be moved into separate files if they grow large. Figures should be referenced by registered file paths only.
