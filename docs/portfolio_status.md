# Portfolio Status — Cipher Topology Lab

## Current status

`cipher-topology-lab` has moved from execution repair into thesis/manuscript scaffolding and planned experiment expansion.

The project now has a defensible empirical core for a diagnostic and reproducibility thesis:

- Six-condition 64-replicate internal TDA evidence run.
- Eight-condition weak-control sensitivity extension.
- Alternate-seed weak-control robustness check.
- Evidence-register traceability for artifact scale, validation status, backend status, external-test status, registered figures, and registered logs.

The next empirical expansion is registered as a staged plan rather than as an open-ended search. The expansion tests seed stability, replicate-scale stability, embedding sample-size sensitivity, and optional external randomness completion.

## Current thesis-ready claim

A conservative portfolio-ready statement is:

> This project develops a reproducible, object-oriented computational workflow for testing whether persistent-homology summaries can serve as auxiliary diagnostics for structure in generated byte streams. Across registered internal runs, deliberately structured weak controls separate from a SHA-256 seeded deterministic baseline under byte-pair and sliding-window embeddings, while AES-CTR, OS CSPRNG, and xorshift32 do not show comparable top-ranked separation. The result is diagnostic and reproducibility evidence, not cryptanalysis, vulnerability discovery, or security certification.

## Recommended portfolio blurb

`cipher-topology-lab` is a reproducible research project that applies persistent homology to generated byte-stream diagnostics. It implements an object-oriented workflow for stream generation, embedding, topological feature extraction, validation, effect-size analysis, evidence registration, and LaTeX manuscript production. Registered 64-replicate experiments show that deliberately structured weak controls separate from a SHA-256 seeded baseline under configured TDA summaries, with the pattern reproduced under an alternate seed. The project is framed as diagnostic and reproducibility infrastructure, not cryptanalysis.
