# Contribution Boundary

## Purpose

Define the public-safe contribution boundary for the cipher-topology diagnostic project.

## Current framing

This project evaluates whether persistent-homology features can serve as auxiliary diagnostics for structure, bias, or control-condition differences in generated byte streams.

## In scope

- Reproducible stream generation
- Null and control-condition design
- Topological feature extraction
- Randomness-test comparison
- Diagnostic reporting
- Evaluation infrastructure

## Out of scope

- Cipher breaking
- Exploit development
- Claims of practical cryptanalytic attack
- Operational security bypass claims
- Undocumented or non-reproducible benchmark claims

## Required before stronger claims

1. Freeze corpus and controls.
2. Document null model.
3. Add baseline notebooks or scripts.
4. Add tests for input generation and output schemas.
5. Produce reproducible result tables and figures.

## Public language

Use diagnostic, benchmark, evaluation, and reproducibility language. Avoid exploit or cryptanalysis language unless a future validated result warrants a separately reviewed claim.
